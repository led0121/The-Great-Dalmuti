# 위대한 달무티 (The Great Dalmuti) - 웹 게임

이 프로젝트는 보드게임 "위대한 달무티"를 웹 브라우저에서 즐길 수 있도록 구현한 멀티플레이어 게임입니다. Node.js와 React를 사용하여 개발되었으며, Socket.io를 통해 실시간 플레이를 지원합니다.

## 🎮 주요 기능

*   **멀티플레이어 지원**: 3명에서 최대 8명까지 함께 플레이할 수 있습니다.
*   **방 시스템**: 방을 생성하고 코드를 공유하여 친구들과 비공개 게임을 즐길 수 있습니다.
*   **실시간 채팅**: 게임 내에서 자유롭게 위치를 옮길 수 있는 채팅창을 제공합니다.
*   **자동화된 규칙**:
    *   **카드 내기**: 더 낮은 계급(더 좋은 숫자)의 카드를 내야 합니다.
    *   **수량 규칙**: 이전 사람이 낸 카드 수보다 **같거나 더 많은** 수의 카드를 낼 수 있습니다.
    *   **조커**: 조커(13)는 어떤 카드로든 사용할 수 있는 만능 카드입니다.
    *   **혁명**: 조커 2장을 가진 플레이어는 대혁명을 일으켜 순위를 뒤바꿀 수 있습니다.
*   **고급 게임 진행**:
    *   **세금 (Taxation)**: 라운드가 시작될 때 꼴등(농노)은 1등(달무티)에게 가장 좋은 카드 2장을 자동으로 바칩니다. 1등은 원하는 카드 2장을 돌려줍니다.
    *   **시장 (Market)**: 세금 징수가 끝나면 시장이 열립니다. 플레이어들은 카드를 시장에 내놓고 다른 플레이어와 무작위로 교환할 수 있습니다.
*   **편의 기능**:
    *   낼 수 없는 카드는 회색으로 표시됩니다.
    *   턴 제한 시간(기본 30초)이 지나면 자동으로 패스됩니다.
    *   **접속 끊김 대응**: 플레이어 이탈 시 자동으로 자리 선정 카드를 선택하거나 세금 징수 단계를 넘기는 등 게임이 멈추지 않도록 처리합니다.
    *   **UI 최적화**: 내 손패를 좌측 하단에 고정하고 카드 개수를 표시하여 플레이 편의성을 높였습니다.

## 🚀 실행 방법

### 필수 요구 사항
*   Node.js (v18 이상 권장)
*   npm

### 설치 및 실행

1.  **프로젝트 클론 및 이동**
    ```bash
    git clone <repository-url>
    cd dal
    ```

2.  **의존성 설치**
    *   서버와 클라이언트의 패키지를 모두 설치해야 합니다.
    ```bash
    # 서버 패키지 설치
    cd server
    npm install
    
    # 클라이언트 패키지 설치
    cd ../client
    npm install
    ```

3.  **한 번에 실행하기**
    *   프로젝트 루트(`dal/`)에서 제공되는 스크립트를 사용하면 서버와 클라이언트를 동시에 실행할 수 있습니다.
    ```bash
    cd .. # 프로젝트 루트로 이동
    bash start.sh
    ```

4.  **접속**
    *   브라우저를 열고 터미널에 표시된 주소(예: `http://localhost:5173`)로 접속합니다.
    *   같은 네트워크에 있는 다른 기기에서도 접속할 수 있습니다 (방화벽 포트 3000, 5173 개방 필요).

## 🎲 게임 규칙 요약

1.  **목표**: 손에 든 모든 카드를 가장 먼저 없애는 사람이 승리합니다.
2.  **계급**:
    *   **위대한 달무티 (1등)**: 가장 먼저 카드를 털어낸 플레이어. 다음 라운드에서 세금을 걷을 권리가 있습니다.
    *   **위대한 농노 (꼴등)**: 가장 마지막까지 카드를 가진 플레이어. 다음 라운드에서 세금을 바쳐야 합니다.
3.  **진행**:
    *   선 플레이어가 카드를 냅니다 (예: 계급 10 카드 2장).
    *   다음 플레이어는 **더 낮은 계급**(예: 9 이하)의 카드를 **같거나 더 많은 수량**(2장 이상)으로 내야 합니다.
    *   낼 카드가 없거나 전략상 내기 싫다면 **패스**할 수 있습니다.
    *   모든 사람이 패스하여 다시 마지막으로 카드를 낸 사람에게 차례가 돌아오면, 그 사람이 새로운 카드를 낼 수 있습니다 (바닥 클리어).

## 🛠 기술 스택

*   **Frontend**: React, Vite, Tailwind CSS, Framer Motion (애니메이션)
*   **Backend**: Node.js, Express, Socket.io
*   **ETC**: Concurrently (동시 실행 관리)
